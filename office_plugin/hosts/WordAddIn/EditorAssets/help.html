LaTeXSnipper Office 插件

在 Word 和 PowerPoint 中插入、编辑和维护由 LaTeXSnipper 管理的原生公式。

系统要求

组件要求
Windows 版本Windows 10 22H2 或更高版本 / Windows 11
Office 版本
  • Microsoft 365 应用（当前通道或月度企业通道）
  • Office 2024 / 2021 / 2019 零售版或批量许可版
  • Office LTSC 2024 / 2021
运行时.NET Framework 4.8；Microsoft Edge WebView2 Runtime（Windows 11 已内置；M365 会自动安装）
Office 体系仅支持 32 位和 64 位桌面版 Office；不支持网页版、移动版和 macOS 版 Office
LaTeXSnipper 桌面端需在本机运行并开启 Office 插件功能，插件通过 127.0.0.1:28765 与本机 Bridge 通信

快速开始

1. 启动桌面端

打开 LaTeXSnipper 桌面端，在设置中启用 Office 插件功能。

2. 打开编辑器

Ribbon 插入按钮会打开空白 MathLive 编辑器，不会默认填入欧拉公式，也不会主动聚焦输入区切换输入法。

3. 插入到文档

默认使用 OLE 公式对象插入。侧边栏只负责独立草稿和新插入；更新公式只从编辑器窗口提交。

如果请求失败，请确认桌面端正在运行且 Office 插件功能已开启。

编辑器快捷键

插入模式按 Enter 插入，更新模式按 Enter 更新；Ctrl+Enter 在 MathLive 输入区内换行。Esc 不关闭编辑器，只收回 MathLive 虚拟键盘。提交期间按钮会禁用；如果 Office 正在执行加载、删除、重编号或 OCR 等命令，编辑器会提示稍后再试，不排队等待。

Word 功能

OLE 与 OMML

默认使用 OLE 公式对象，公式显示由 MathJax SVG 转为 EMF+Dual 矢量预览。设置中可切换到 Word OMML。

加载、更新和删除

选中受管理公式后用 Ribbon 加载到编辑器修改。编辑器提交后更新原公式；侧边栏不会承担更新入口。删除同时移除公式和元数据。编辑器提交、加载所选、删除、重编号、OCR 等 Office 命令互斥执行，正在处理时不会排队堆积新的提交。

侧边栏草稿

侧边栏是独立的新公式草稿区。加载公式到编辑器时会临时显示该公式属性；编辑器提交或取消后恢复原来的侧边栏草稿。

自动编号与重编号

自动编号只对未编号的行间公式生效，不会把行内公式转换为编号公式。重编号按文档顺序更新自动编号文本，不重新渲染公式。编号位置设置只影响新插入的编号公式和之后自动编号的普通行间公式。

带编号公式布局

带编号公式不使用 Word 表格。公式和编号位于同一个普通段落中：公式通过居中制表位对齐，编号按设置放在最左或最右。点击公式或编号都可以加载、更新或删除同一个公式。

OLE 对象尺寸

Word OLE 公式使用固定渲染基线。更新已有 OLE 公式时保留用户在 Word 中手动拉伸出的对象倍率。

截图识别

点击后等待桌面端截图结果，再次点击可取消。

PowerPoint 功能

OLE 与 PNG 图片

默认使用 OLE 公式对象，公式显示由 MathJax SVG 转为 EMF+Dual 矢量预览。设置中可切换到 PNG 图片方式。

加载和删除

选中受管理公式后用 Ribbon 加载到同一套 MathLive 编辑器修改。编辑器提交后替换原位置公式，并保留用户在幻灯片中手动拉伸出的对象倍率。删除同时移除形状和元数据。编辑器提交、加载所选、删除和 OCR 等 Office 命令互斥执行，正在处理时不会排队堆积新的提交。

侧边栏草稿

侧边栏是独立的新公式草稿区，不负责更新已有公式。

OLE 对象尺寸

PowerPoint OLE 公式使用固定渲染基线。更新已有 OLE 公式时保留用户在幻灯片中手动拉伸出的对象倍率。

截图识别

点击后等待桌面端截图结果，识别后的 LaTeX 自动填入编辑器。再次点击可取消。

插件架构

项目实现
运行方式Windows 原生 VSTO 插件，Ribbon、侧边栏、设置和帮助页均在 Office 内打开。
Word 公式默认写入 LaTeXSnipper OLE 公式对象；OMML 是可选插入后端。公式附带 LaTeX 源码、显示模式、编号模式等管理元数据。
PowerPoint 公式默认写入 LaTeXSnipper OLE 公式对象；PNG 图片是可选插入后端。元数据写入形状标签（shape.Tags）。
连接方式HTTP 连接本机 Bridge（127.0.0.1:28765），LaTeX 转换和截图识别均由桌面端完成。
编辑器WebView2 托管的 MathLive 编辑器，支持可视化编辑和实时预览。插入模式默认空白打开，不主动聚焦 MathLive 输入区；编辑器提交与 Office 命令统一串行化。

边界说明

  • 只有由 LaTeXSnipper 插入并保留元数据的公式才能被插件管理。
  • 普通 Office 公式没有 LaTeXSnipper 元数据，插件不会猜测其 LaTeX 来源。
  • 复制粘贴后元数据丢失的公式无法继续维护。
  • 如果加载项未显示，请关闭 Office，重新运行安装包进行修复或重新安装，然后再打开 Office。

LaTeXSnipper Office Plugin

Insert, edit, and maintain native formulas managed by LaTeXSnipper in Word and PowerPoint.

Office
  • Microsoft 365 Apps (Current or Monthly Enterprise Channel)
  • Office 2024 / 2021 / 2019 Retail or Volume License
  • Office LTSC 2024 / 2021

Quick Start

1. Start the client

Open LaTeXSnipper and enable Office plugin in settings.

2. Open the editor

Ribbon insert buttons open a blank MathLive editor. The editor no longer preloads Euler's identity or forces focus into the input area.

3. Insert into Office

OLE formula objects are the default. The side pane owns an independent draft for new insertions; formula updates are submitted only from the editor window.

If requests fail, make sure the desktop client is running and Office plugin is enabled.

Editor Shortcuts

Enter inserts in insert mode and updates in update mode; Ctrl+Enter keeps its MathLive line-break behavior. Esc does not close the editor; it only retracts the MathLive virtual keyboard. Submit buttons are disabled during submission. If Office is already running load, delete, renumber, OCR, or another command, the editor reports that processing is busy instead of queueing the submit.

Word Features

OLE and OMML

OLE formula objects are the default. MathJax SVG output is converted to an EMF+Dual vector preview. Word OMML remains available in settings.

Load, Update, Delete

Use the Ribbon to load a managed formula into the editor. Submitting the editor updates the original formula; the side pane is not an update entry point. Deletion removes both formula and metadata. Editor submit, load selected, delete, renumber, and OCR Office commands run through a single command gate; a busy command rejects new editor submits instead of queueing them.

Side Pane Draft

The side pane is an independent new-formula draft. Loading a formula into the editor temporarily shows that formula's properties; after editor accept or cancel, the original side pane draft is restored.

Auto Numbering & Renumber

Auto Number only applies to unnumbered display equations, so inline equations are never converted into numbered formulas. Renumber updates automatic numbers in document order without re-rendering. Number placement affects newly inserted numbered formulas and ordinary display formulas numbered later.

Numbered Formula Layout

Numbered formulas do not use Word tables. The formula and number live in a normal paragraph: the formula is aligned by a center tab stop, and the number is placed at the left or right edge according to settings. Clicking either the formula or the number can load, update, or delete the same formula.

OLE Object Size

Word OLE formulas use a fixed rendering baseline. Updating an existing OLE formula preserves the object stretch factor manually set in Word.

PowerPoint Features

OLE and PNG Image

OLE formula objects are the default. MathJax SVG output is converted to an EMF+Dual vector preview. PNG image insertion remains available in settings.

Load and Delete

Use the Ribbon to load a managed formula into the same MathLive editor. After editing, the new formula replaces the original at the same position and preserves the object stretch factor manually set on the slide. Deleting removes both shape and metadata. Editor submit, load selected, delete, and OCR Office commands run through a single command gate; a busy command rejects new editor submits instead of queueing them.

Side Pane Draft

The side pane is an independent new-formula draft and is not an update entry point.

OLE Object Size

PowerPoint OLE formulas use a fixed rendering baseline. Updating an existing OLE formula preserves the object stretch factor manually set on the slide.

Plugin Architecture

AreaImplementation
RuntimeNative Windows VSTO add-in. Ribbon, side pane, settings, and help open inside Office.
Word OutputLaTeXSnipper OLE formula objects by default; OMML is an optional backend. Metadata stores LaTeX source, display mode, and numbering mode.
PowerPoint OutputLaTeXSnipper OLE formula objects by default; PNG image insertion is an optional backend. Metadata is stored in shape tags.
ConnectionHTTP to local Bridge at 127.0.0.1:28765; LaTeX conversion and OCR run on the desktop client.
EditorWebView2-hosted MathLive editor with visual editing and live preview. Insert mode opens blank and does not force focus into the MathLive input area; editor submissions are serialized with Office commands.

Boundaries

  • Only formulas inserted by LaTeXSnipper with metadata can be managed by the plugin.
  • Plain Office equations lack LaTeXSnipper metadata; the plugin does not guess their LaTeX source.
  • Formulas that lose metadata after copy-paste cannot be managed.
  • If the add-in does not appear, close Office, run the installer again to repair or reinstall the add-in, then reopen Office.
